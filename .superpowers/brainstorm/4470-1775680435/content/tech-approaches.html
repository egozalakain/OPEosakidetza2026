Enfoque Tecnológico — OPE Quiz App

3 opciones con diferentes trade-offs. Haz clic en la que prefieras.

A

Next.js + Supabase (Vercel)

Stack: React/Next.js frontend + Supabase (PostgreSQL + Auth)

Hosting: Vercel (gratis) + Supabase free tier (500MB DB)

Pros

  • Ecosistema maduro, muchos componentes listos
  • Auth integrada en Supabase (sin código extra)
  • Despliegue automático con git push
  • Charts/gráficos fácil con Recharts
  • Gratis para uso personal

Cons

  • Más pesado (React + framework)
  • Supabase free tier puede dormir la DB tras inactividad
  • Dos servicios que gestionar
B

Astro + Cloudflare D1 (Cloudflare Pages)

Stack: Astro (con islas React para interactividad) + D1 (SQLite en edge)

Hosting: Cloudflare Pages (gratis) + D1 free tier (5GB)

Pros

  • Ultrarrápido: páginas estáticas + interactividad donde hace falta
  • D1 no duerme — siempre disponible, 5GB gratis
  • Todo en un solo proveedor (Cloudflare)
  • SQLite es simple y predecible

Cons

  • Astro menos popular que Next.js
  • D1 aún relativamente nuevo
  • Auth hay que implementarla manual
C

Next.js Full-Stack (Vercel + Vercel Postgres)

Stack: Next.js con Server Actions + Vercel Postgres (Neon)

Hosting: Vercel (gratis) + Vercel Postgres free tier (256MB)

Pros

  • Todo integrado: un solo framework, un solo proveedor
  • Server Actions = API sin boilerplate
  • ORM con Drizzle/Prisma para esquema tipado
  • Despliegue git push, cero config
  • Gran comunidad y documentación

Cons

  • 256MB en free tier (suficiente para 200 preguntas)
  • Cold starts en serverless

💡 Mi recomendación: Opción C

Next.js full-stack con Vercel Postgres es la opción más equilibrada: un solo proveedor, despliegue trivial, framework maduro con Server Actions (sin API separada), y 256MB de DB es más que suficiente para 200 preguntas + tu historial de exámenes. Tailwind CSS para el diseño y Recharts para los gráficos.
