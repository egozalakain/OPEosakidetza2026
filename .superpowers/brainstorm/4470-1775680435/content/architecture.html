Modelo de Datos (5 tablas)

Esquema de Base de Datos

📋 questions

  id (PK)

  number (1-200)

  topic (ej: "Ley 44/2003")

  text (enunciado)

  option_a, option_b, option_c, option_d

  correct_answer (a/b/c/d)

  explanation (justificación)


📝 exams

  id (PK)

  started_at, finished_at

  mode (exam | study)

  timer_mode (countdown | stopwatch | none)

  timer_seconds (límite si countdown)

  correction_mode (immediate | end)

  question_selection (random | weak | topic)

  topic_filter (nullable)

  total_questions

  score, correct_count, wrong_count, blank_count

  penalized_score

✏️ exam_answers

  id (PK)

  exam_id (FK → exams)

  question_id (FK → questions)

  selected_answer (a/b/c/d | null)

  is_correct (boolean)

  flagged (boolean - "no estoy seguro")

  time_spent_seconds


📊 question_stats

  question_id (PK, FK → questions)

  times_shown

  times_correct

  times_wrong

  times_blank

  error_rate (calculado)

  last_answered_at


⚙️ user_settings

  key (PK)

  value

  (preferencias: tema, idioma, etc.)
